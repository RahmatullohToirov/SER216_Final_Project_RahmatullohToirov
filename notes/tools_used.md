# Tools Used – Digital Parking Permit System

## Overview

This document lists all the tools used to complete the SER216 Final Project and explains their role in the workflow.

---

## Tool List

### 1. Google Docs
- **Purpose:** Writing and formatting the final project report
- **Usage:** Tasks 1 (Requirements), Task 4 (Defect Analysis), and overall report narrative
- **Why chosen:** Easy to format, export to PDF, and share with instructors

### 2. Google Sheets
- **Purpose:** Creating the test case table (Task 3)
- **Usage:** Six test cases organized with columns: Test ID, Feature, Input/Data, Expected Result, Test Type
- **Why chosen:** Tabular format is ideal for test case documentation; easy to export as CSV or XLSX

### 3. diagrams.net (draw.io)
- **Purpose:** Drawing the workflow diagram (Task 2)
- **Usage:** Created the end-to-end flow: Vehicle Registration → Permit Request → Zone Check → Permit Approval → Parking Verification
- **Why chosen:** Free, browser-based, supports flowchart and UML notation, exports to PNG

### 4. GitHub
- **Purpose:** Version control, file hosting, and project evidence
- **Usage:**
  - Created repository: `SER216_Final_Project_RahmatullohToirov`
  - Added README.md with project description
  - Uploaded all project files
  - Created feature branch: `feature/upload-project-files`
  - Made meaningful commits at each stage
  - Created and merged pull request
- **Why chosen:** Industry-standard version control platform; required by the project specification

## Workflow Summary

```
1. Plan project structure
        ↓
2. Write requirements (Google Docs)
        ↓
3. Draw workflow diagram (diagrams.net)
        ↓
4. Create test case table (Google Sheets)
        ↓
5. Perform defect analysis (Google Docs)
        ↓
6. Compile final PDF report
        ↓
7. Upload to GitHub with proper commits
        ↓
8. Create feature branch and pull request
        ↓
9. Merge pull request and submit
```
